## üõë Challenges & Lessons Learned

| Challenge | Impact | Lesson Learned |
| :--- | :--- | :--- |
| **CSS Migration Complexity** | Legacy `clamp()` values and specificities required extra time to map correctly. | **Explicit Documentation**: Documenting the font-stack and token hierarchy upfront prevents circular reference bugs. |
| **Visual Snapshot Churn** | Mobile baselines required frequent updates as design adapted (e.g., hamburger menu). | **Protocol Needed**: We need a clear process for *when* to update snapshots to avoid "blindly updating" vs. "catching regressions." |
| **Fidelity vs. Usability** | Strict desktop match vs. mobile usability (e.g., nav menu) required distinct NFRs. | **Explicit ACs**: Defining separate NFRs (Visual Fidelity for Desktop, Usability for Mobile) is the winning pattern. |

## üí° Key Insights

1. **Visual Fidelity NFRs Work**: The "0% Desktop Regression" rule was not just a constraint but a quality gate that forced discipline.
2. **AI as Safety Net**: AI reviews consistently caught "invisible" debt (missing aria attributes, correct token usage) that human reviewers might miss in speed.
3. **Design Intelligence**: Including "Design Intelligence" tables in stories (mapping legacy tokens to new variables) minimized developer guesswork.
